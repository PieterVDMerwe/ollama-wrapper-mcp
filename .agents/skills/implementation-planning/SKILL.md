# Implementation Plan Construction

Use this skill when drafting, updating, or modifying the project's `implementation_plan.md` file.

## 1. Plan Structure & Phasing
Every implementation plan must divide the project work into clear, sequential phases:
* **Phase-Based Breakdown**: Do not list a flat checklist of tasks. Group them into progressive phases (e.g., *Phase 1: Basic Setup*, *Phase 2: Core Functionality*, *Phase 3: Integration & Security*, *Phase 4: Testing & Verification*).
* **Clear Phase Gates**: Define what constitutes the completion of each phase before the next begins.

## 2. Managing Updates and Modifications
* **Incremental Phases**: When the user requests a new feature or modifications to the existing design, update the implementation plan by appending or inserting new phases or sub-phases.
* **Traceability**: Clearly note which phase/step addresses which component of the user's requirements.
* **Approval Flow**: Ensure the implementation plan is approved by the user before executing the changes.
